# **Pal-Lock Engine – Secure Trust-Based Matching Algorithm (v1)**

---

# **1. Core Philosophy**

> Pal-Lock is NOT a normal search engine.
> It is a **trust-prioritized, security-aware, human network matching system**.

Every query must satisfy:

1. **Trust First**
2. **Relevance Second**
3. **Distance Third**
4. **Security Always**

---

# **2. High-Level Flow**

```
INPUT → VALIDATE → CLASSIFY → SEARCH → FILTER → RANK → SECURE → RETURN
```

---

# **3. Step-by-Step Algorithm Design**

---

## **STEP 1: INPUT LAYER**

User submits:

```json
{
  "user_id": "U1",
  "query": "mechanic",
  "image": null,
  "location": { "lat": 32.77, "lng": -96.79 },
  "visibility_scope": "auto"
}
```

---

## **STEP 2: SECURITY VALIDATION (CRITICAL 🔒)**

Before ANY search happens:

### **Checks**

* JWT Authentication (user identity verified)
* Rate limiting (prevent spam/abuse)
* Input sanitization (prevent injection attacks)
* Image scanning (malware / harmful content)
* Permission check (what data user is allowed to see)

### **Pseudo**

```pseudo
if !isAuthenticated(user):
    reject("Unauthorized")

if isRateLimited(user):
    reject("Too many requests")

sanitize(query)
validate(image)
```

---

## **STEP 3: QUERY CLASSIFICATION**

Determine intent:

* Service (mechanic, plumber)
* Product (TV, laptop)
* Help (ride, assistance)

### **AI/NLP Layer**

```pseudo
if contains(service_keywords):
    type = "service"

if image_present:
    type = "visual_search"
```

---

## **STEP 4: TRUST LAYER SEARCH**

Search happens in **3 stages**:

---

### **LAYER 1: FRIENDS (Highest Trust)**

```sql
SELECT * FROM dock_items, services
WHERE user_id IN (user_contacts)
```

---

### **LAYER 2: COMMUNITIES**

```sql
SELECT * FROM dock_items, services
WHERE user_id IN (community_members)
```

---

### **LAYER 3: PUBLIC (Fallback)**

```sql
SELECT * FROM dock_items, services
WHERE visibility = 'public'
```

---

## **STEP 5: DEEP SEARCH SOURCES**

Search across:

* `dock_items` (products/services)
* `services` (structured listings)
* `stores`
* `posts` (content-based discovery)

---

## **STEP 6: FILTERING ENGINE**

Apply strict filters:

* Location proximity
* Availability (online/active)
* Visibility permissions
* Blocked users removed
* Community restrictions

```pseudo
results = results.filter(
  not_blocked &&
  within_visibility &&
  within_radius
)
```

---

## **STEP 7: TRUST + RELEVANCE RANKING (CORE LOGIC)**

Each result gets a **score**:

```
FINAL SCORE =
(Trust Score * 0.4) +
(Relevance Score * 0.3) +
(Distance Score * 0.2) +
(Activity Score * 0.1)
```

---

### **Trust Score**

* Friend → 1.0
* Same community → 0.7
* Public → 0.4

---

### **Relevance Score**

* Keyword match
* Tag match
* Image similarity

---

### **Distance Score**

* Closer = higher score

---

### **Activity Score**

* Recently active users rank higher

---

## **STEP 8: SECURITY FILTER (VERY IMPORTANT 🔒)**

Before returning results:

### **Privacy Enforcement**

* Respect Dock visibility:

  * friends only
  * community only
  * public

### **Sensitive Data Protection**

* Hide:

  * phone numbers
  * exact address
* Show only:

  * profile preview
  * approximate location

---

### **Anti-Abuse System**

* Detect spam accounts
* Detect fake services
* Down-rank suspicious users

```pseudo
if user_flagged:
    reduce_score()
```

# **7. SIMPLE PSEUDOCODE (FULL FLOW)**

```pseudo
function palLockSearch(user, query):

    authenticate(user)

    intent = classify(query)

    results = []

    results += searchFriends(user, query)
    if results.empty:
        results += searchCommunities(user, query)
    if results.empty:
        results += searchPublic(query)

    results = filter(results)

    scored = rank(results)

    secured = applyPrivacy(scored)

    return secured
```
